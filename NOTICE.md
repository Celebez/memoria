**Effective from initial publication.**

project. The documentation inside it is a finished work intended to be
read, shared, and cited — not modified, remixed, or forked into new
versions.

The only authoritative version of this repository lives at:

### Contributions are not accepted

- **Issues** are disabled on the canonical repository.
- **Discussions** are disabled.
- **Pull requests** will be closed without action.
- **Wiki** is disabled.

If you spot a typo or inaccuracy, the correct path is to cite the
canonical version with a note like:

> "Per the canonical Memoria docs at https://github.com/Celebez/memoria
> (accessed YYYY-MM-DD)..."

This keeps the work attributable to a single source of truth.

### Forking is permitted, with restrictions

You may fork the repository for **personal reading, study, and
archival**. However, per `LICENSE-DOCS`:

- You **may not** redistribute the forked documentation as if it were

- You **may not** publish modifications, translations, or

## Why this policy

Memory and decision-explanation systems are safety-critical
infrastructure. Documentation for them must be:

1. **Authoritative** — readers need to know which version is "the
   truth". Multiple divergent forks create confusion and risk.
2. **Stable** — links shared today must resolve to the same content
   tomorrow. Forks that diverge create broken references.
3. **Attributable** — the author and source must be unambiguous so
   readers can verify the provenance of advice they follow.

A single canonical version, with clear license terms, achieves all
three.
